AI Translation Project

Comparing ChatGPT, DeepL, and IBM Watsonx for multilingual e-learning

Overview

This project developed an end-to-end AI translation workflow to evaluate how well leading AI tools handle multilingual e-learning content. The goal: determine which platform can reliably translate a full course into multiple languages while preserving instructional quality, terminology accuracy, and learner experience.

The workflow batch-translated a complete e-learning course into 9 languages in under 30 minutes, using API scripts built in Python. A volunteer evaluation framework was designed to assess translation quality across dimensions like fluency, accuracy, and cultural appropriateness.

Tools Compared

ChatGPT (GPT-4)

OpenAI's large language model accessed via API. Strong contextual understanding and ability to handle nuanced educational content. Supports prompt engineering for domain-specific translation instructions.

GPT-4 API Python

DeepL

Specialized neural machine translation service known for high-quality European language translations. API-based with glossary support for consistent terminology across documents.

DeepL API Glossary Support

IBM Watsonx

IBM's enterprise AI platform with language translation capabilities. Evaluated for integration potential within IBM SkillsBuild's existing infrastructure and enterprise compliance requirements.

Watsonx Enterprise

Methodology

  1. Content Selection — Selected a representative e-learning course with varied content types: instructional text, quiz questions, UI labels, and multimedia descriptions.
  2. API Script Development — Built Python scripts to batch-translate content through each platform's API, handling rate limits, error recovery, and output formatting.
  3. Multi-language Translation — Translated the full course into 9 target languages, measuring speed, cost, and completeness for each tool.
  4. Quality Evaluation Framework — Designed a volunteer-based evaluation rubric assessing fluency, accuracy, terminology consistency, and cultural appropriateness.
  5. Comparative Analysis — Compiled results into a recommendation matrix with scoring across quality, speed, cost, and integration criteria.

Key Findings

GPT-4 excelled at contextual understanding and handling ambiguous educational content, but required careful prompt engineering to maintain consistency across long documents.

DeepL delivered the most polished translations for European languages with minimal post-editing needed. Glossary features ensured terminology consistency.

IBM Watsonx offered the strongest enterprise integration path and compliance features, making it the best fit for organizations with existing IBM infrastructure.

Recommendations

Criteria Best Tool Notes
Translation Quality DeepL Especially strong for European languages
Contextual Understanding GPT-4 Best at handling nuanced instructional content
Enterprise Integration Watsonx Seamless with IBM ecosystem
Speed / Throughput DeepL Fastest batch processing times
Cost Efficiency DeepL Best value per character translated
Customizability GPT-4 Prompt engineering allows domain-specific tuning

Impact

This research informed IBM SkillsBuild's approach to multilingual content delivery, providing a data-driven framework for selecting translation tools based on language pair, content type, and organizational constraints. The batch-translation scripts reduced what would have been weeks of manual work into a 30-minute automated process.
